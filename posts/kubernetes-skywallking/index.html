    kubernetes-离线部署Skywallking

    April 7, 2021 4 min read kubernetes kubernetes skywalking
    离线状态的环境下如何去部署skywalking
    On this page
    • 在线部署chartmuseum
    • 下载Skywalking包
      • 添加elasticsearch仓库
    • 上传本地Helm
    • 部署skywalking
      • 准备离线镜像
    • Helm中的Elasticsearch可能会存在问题

    注意：请各位记住把所有离线包全拿到本地…

    在线部署chartmuseum

    直接使用最简单的 docker run 方式，使用local 本地存储方式，通过 -v 映射到宿主机 /opt/charts 更多支持安装方式见官网

    1mkdir /opt/charts
2docker run -d \
3  -p 8080:8080 \
4  -e DEBUG=1 \
5  -e STORAGE=local \
6  -e STORAGE_LOCAL_ROOTDIR=/charts \
7  -v /opt/charts:/charts \
8  chartmuseum/chartmuseum:latest

    下载Skywalking包

    1git clone https://github.com/apache/skywalking-kubernetes.git
2# 更换仓库
3cd skywalking-kubernetes-master/chart/skywalking/
4vim Chats.yaml
5dependencies:
6  - name: elasticsearch
7    version: ~7.12.1  # 官网的版本号为7.5.1 最新的elastic版本为7.12.1
8    repository: http://localhost:8080 # 修改为你本地的Repo地址
9    condition: elasticsearch.enabled

    添加elasticsearch仓库

    1helm repo add elastic https://helm.elastic.co
2helm pull elastic/elasticsearch  # 把elasticsearch内容拉下来

    上传本地Helm

    以防万一请先安装helmpush插件

    https://github.com/chartmuseum/helm-push

    1helm repo add chartmuseum http://localhost:8080
2curl --data-binary "@elasticsearch-7.12.1.tgz" http://localhost:8080/api/charts
3helm push /root/skywalking-kubernetes-master/chart/skywalking/ chartmuseum
4helm repo update # 更新仓库

    你可以尝试搜索一下

    保证仓库中存在elasticsarch和skywalking

    1[root@k-master1 ~]# helm search repo
2NAME                            CHART VERSION   APP VERSION     DESCRIPTION                                       
3chartmuseum/elasticsearch       7.12.1          7.12.1          Official Elastic helm chart for Elasticsearch     
4chartmuseum/skywalking          4.0.0                           Apache SkyWalking APM System

    部署skywalking

     1cd skywalking-kubernetes/chart
 2helm dep up skywalking
 3# change the release name according to your scenario
 4export SKYWALKING_RELEASE_NAME=skywalking
 5# change the namespace according to your scenario
 6export SKYWALKING_RELEASE_NAMESPACE=default
 7helm install "${SKYWALKING_RELEASE_NAME}" skywalking -n "${SKYWALKING_RELEASE_NAMESPACE}" \
 8  --set oap.image.tag=8.1.0-es7 \
 9  --set oap.storageType=elasticsearch7 \
10  --set ui.image.tag=8.1.0 \
11  --set elasticsearch.imageTag=7.5.1
12helm uninstall skywalking # 卸载skywalking

    准备离线镜像

    1busybox:1.30
2docker.elastic.co/elasticsearch/elasticsearch:7.5.1
3apache/skywalking-oap-server:8.1.0-es7
4apache/skywalking-ui:8.1.0
5chartmuseum/chartmuseum:latest
